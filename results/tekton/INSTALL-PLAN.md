# Tekton Pipelines - Installation Plan

## Project Overview
Tekton Pipelines is a Kubernetes-native CI/CD framework. It runs a controller in `tekton-pipelines` namespace that processes TaskRun and PipelineRun CRDs.

## Installation Method
- **Method**: `kubectl apply` with official release YAML
- **Version**: v1.11.1
- **Release URL**: `https://storage.googleapis.com/tekton-releases/pipeline/previous/v1.11.1/release.yaml`
- **Namespace**: `tekton-pipelines` (created by release YAML)

## Telemetry Configuration

### Metrics
- Configure `config-observability` ConfigMap to export via OTLP gRPC to the OTel collector
- Endpoint: `otel-collector-opentelemetry-collector.opentelemetry.svc.cluster.local:4317`
- Protocol: `grpc`

### Traces
- Configure tracing in same ConfigMap
- Protocol: `grpc`
- Endpoint: same OTel collector
- Sampling rate: `1.0` (100%)

### Logs
- Container stdout only (no OTLP log export available)
- OTel collector k8s log collection not configured (out of scope)

## Collector Changes
- Add Prometheus scrape job for `tekton-pipelines` namespace (port 9090) as fallback verification
- Primary: OTLP push from Tekton controller

## Routing/Ingress
- Not applicable: Tekton is a CI/CD system, not an HTTP proxy
- Traffic is generated by creating Kubernetes CRDs (TaskRun, PipelineRun)

## Traffic Generation
1. Create simple TaskRuns that run `echo` commands
2. Create PipelineRuns with multiple tasks
3. Include some failing runs to exercise error metrics
4. Run 10+ TaskRuns and 5+ PipelineRuns

## Steps
1. Install Tekton v1.11.1 via release YAML
2. Wait for controller to be ready
3. Patch `config-observability` to enable OTLP traces + metrics
4. Update OTel collector to scrape Prometheus metrics from controller
5. Create TaskRuns and PipelineRuns
6. Verify telemetry flowing to `/tmp/otel-eval-tekton/`
